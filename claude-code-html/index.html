工作方式

我的工作方式：先把边界讲清楚，再把流程做稳定。

我习惯先梳理目标、风险、依赖和验收标准，再推进实现。遇到重复问题，会优先沉淀文档、脚本、检查清单或自动化流程，减少团队后续的沟通和排障成本。

01理解目标先确认业务目的和交付边界
02识别风险提前看环境、权限、数据和依赖
03整理流程把操作步骤变成可复用清单
04自动检查用脚本和 CI 减少低级错误
05记录经验把排障和决策沉淀到知识库
06持续优化让下一次交付更快、更稳
